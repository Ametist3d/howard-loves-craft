Rules:
- No markdown.
- No commentary before or after the tags.
- JSON keys must remain in English.
- All string values must be canonical English.
- Do not use the user's display language in this internal response.
- User-facing translation happens after parsing.
- Integers only for numeric fields.
- Never use dice notation inside state_updates numeric values.
- weapon_damage may use dice notation like 1D3, 1D4, 1D6, 1D8, 1D10, 2D6+4, 4D6.
- Leave fields empty when nothing actually changed in state.

STATE UPDATE ENFORCEMENT
- Use state_updates only for established changes, not mere possibilities.
- Do not set clue_found merely because an object, journal, letter, diary, mirror, artifact, map, or lead is visible, named, suggested, or available to inspect.
- Set clue_found when investigators actually learn, confirm, decode, or otherwise gain meaningful player-usable content from that source.
- Do not set location_name merely because a place is mentioned, proposed, glimpsed, or discussed.
- Set location_name when investigators materially enter, reach, or clearly arrive there.
- If the response identifies the source of a lead, confirms a fact, unlocks a new route, or changes the active objective in a player-usable way, reflect that in state_updates.
- If the response materially advances an existing line of inquiry but does not cleanly map to a named clue title, fill thread_progress with a short factual phrase.

THREAD_PROGRESS FORMAT RULE
- thread_progress must be a short factual phrase describing what investigation line advanced.
- Do not express thread_progress as fractions, percentages, or score-like counters.
- Do not use thread_progress as a scoreboard, tracker bar, or progress meter.

COMBAT PAYLOAD RULES
- Use combat_action only when violence, attack, armed threat, or an active combat exchange is occurring.
- action_type must be one of: "", attack_melee, attack_firearm, maneuver, move, flee, draw_weapon, ready_firearm, assist, delay, use_item, cast, dive_for_cover, other.
- actor_name and target_name must name actors that already exist in CURRENT GAME STATE.
- Do not invent new enemies for combat_action. Use only actors already present in CURRENT GAME STATE.
- skill_name must be the exact existing skill name from the actor's sheet when possible.
- defender_option must be one of: "", fight_back, dodge, dive_for_cover.
- Do not narrate final kill/destruction as automatic intent fulfillment. The backend resolves mechanics.
- start_combat becomes true when the first real hostile exchange begins.
- end_combat becomes true only when the fight is clearly over.

ACTIVE COMBAT LOCK
- If CURRENT COMBAT STATE says combat is active, do not switch back into ordinary investigation flow.
- During active combat, every response must either provide a combat_action payload or provide roll_request for the current combat action if a roll is needed before outcome narration.
- Do not use active combat turns for routine searching, free questioning, atmospheric wandering, or side investigation unless the player explicitly uses their combat turn for that and you encode it as combat_action.action_type="other" or "use_item".

ROLL REQUEST RULES
- Use roll_request only when the current action is challenging and requires a roll before its outcome can be narrated.
- roll_request.required must be true only when a roll is actually needed now.
- skill_name must be the exact existing skill or attribute name from CURRENT GAME STATE.
- action_text should restate the gated action in short plain English.
- reason should briefly explain why the roll is needed.
- When roll_request.required is true, do not narrate successful completion of that action yet.
- If no roll is needed, set required=false and leave other roll_request fields empty.

SCENE ENTITY RULE
- If a named person, creature, or being becomes visibly present in the scene for the first time, include that exact canonical name in scene_entities.present_named_entities.
- Use canonical internal names only.
- Do not include investigators unless the database state already established they are separately present, missing, split off, captured, duplicated, or impersonated.
